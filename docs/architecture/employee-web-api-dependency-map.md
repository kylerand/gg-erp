# Employee Web API Dependency Map

## Scope and evidence

This map aligns employee web screens/workflows to API dependencies using:

- Web routes and feature API clients in `apps/web/src/app/router.ts` and `apps/web/src/features/*/api.ts`
- API context route/service contracts in `apps/api/src/contexts/*`
- Canonical API operations in `packages/domain/src/model/*`
- Existing contract/failure coverage in `apps/api/src/tests/*.test.ts`

Assumptions and standards posture:

- API contracts are TypeScript-first and versioned by explicit request/response interfaces.
- Clear modularity over cleverness: context services enforce invariants and emit events/audit points; repositories handle persistence/transactions.
- Gap closure is incomplete unless tests cover success and failure paths with audit/event/observability assertions.
- New projection/read contracts that require storage changes must ship with additive migrations before web cutover.

Coverage status legend:

- `вњ… Covered` = contract exists and has direct test coverage
- `вљ пёЏ Partial` = contract exists but misses read/write surface or direct tests
- `вќЊ Gap` = dependency expected by screen/workflow but no current contract

## Highest-priority contract closure sequence

1. **P0:** Deliver missing read contracts required by existing web clients (`/auth/me`, `/customers` list, `/inventory/lots` list, `/planning/work-orders` list, `/reporting/snapshot`).
2. **P0:** Add API integration/contract tests for currently untested ticket-task and attachment mutations.
3. **P1:** Implement workspace/reporting/blocked-alert read models with freshness metadata and stale-state semantics.
4. **P1:** Implement `sop-ojt` context contracts (`GET /sop/*`, `POST /ojt/*`) and wire into work-order completion flows.
5. **P2:** Expand accounting monitor APIs (list/filter/escalation) and admin audit/integration-health surfaces.

## Migration and rollout implications (for contract closure)

| Dependency closure area | Likely persistence impact | Migration expectation |
|---|---|---|
| Workspace summary + blocked-alert/read-model feeds | New projection tables/materialized views keyed by site/role and freshness timestamp | Add additive schema migrations and backfill jobs before enabling dashboard/reporting cutover |
| Inventory reservation/shortage read APIs | Reservation ownership/reason columns or supporting projection table updates | Add non-breaking columns/indexes first; keep old read path until parity checks pass |
| Planning board read APIs | Query-optimized indexes for slot/labor/work-order demand projections | Add indexes in controlled migrations with explain-plan validation and rollback notes |
| Accounting monitor list/filter/escalation | Additional retry-count/escalation metadata on sync records | Add additive fields with default values; release read path before enforcing new escalation policy |
| Admin audit/integration health feeds | Expanded audit/event query surfaces over existing audit tables | Prefer read-side indexes/materialized views over mutating canonical audit schema |
