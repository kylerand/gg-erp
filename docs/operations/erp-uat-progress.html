Useful-State Rule

  • Every card, button, tab, and row routes somewhere useful or performs a visible action.
  • Every filter updates URL state, active state, counts, and empty-state copy.
  • Every empty state tells the operator what data is missing and what to do next.

Reference Model

The target work-order experience follows the Shopmonkey-style command center: customer, cart, services, parts, labor, QC, SOP, messages, accounting, payments, and activity in one workspace.
