The Problem

Why We’re Moving Off ShopMonkey

Limitations of ShopMonkey
  • No custom SOP or workflow enforcement
  • No OJT / tech training integration
  • No build-slot planning or capacity visibility
  • Limited, fragile QuickBooks sync
  • No AI-assisted operations
  • Locked into a vendor roadmap we don’t own
What the New Manufacturing OS Delivers
  • Full work-order & ticket lifecycle ownership
  • SOP library wired to job completion gates
  • Deterministic build-slot planning engine
  • Rock-solid QuickBooks sync (invoices, bills, GL)
  • Amazon Bedrock AI assistant built in from day one
  • 100% owned AWS infrastructure — no vendor lock-in
How It’s Built

System Architecture

Front End
  • Employee Web App (React / Next.js)
  • Floor Technician App (separate Next.js app)
  • Cognito authentication with role-based access
AWS Backend
  • Lambda API — TypeScript, all 8 contexts scaffolded
  • Aurora PostgreSQL Serverless v2
  • EventBridge async event routing between modules
  • Step Functions for migration & planning workflows
8 Operational Modules
  • Identity & Access · Work Orders & Tickets
  • Inventory & Parts Reservations
  • SOP Library & OJT Tracking
  • Build Slot Planning Engine
  • QuickBooks Accounting Integration
  • ShopMonkey Migration Pipeline
  • AI Orchestration (Bedrock RAG + Copilot)
Platform Foundations
  • Full audit log on every mutation with actor + correlation ID
  • Outbox pattern for reliable async event delivery
  • Terraform-managed AWS infrastructure
